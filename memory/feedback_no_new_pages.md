metadata:
  type: feedback

Ne jamais créer de nouveaux fichiers HTML (ni JS/CSS associés) lors d'une modification du site. Éditer uniquement les fichiers existants.

**Why:** L'utilisateur a explicitement demandé d'arrêter ce comportement — les nouvelles pages alourdissent le projet sans que ce soit demandé.

**How to apply:** Toute demande de modification (design, contenu, fonctionnalité) doit se faire en éditant les fichiers déjà présents : index.html, style.css, app.js, shared.js, product.html, product.css, product.js, checkout.html, checkout.css, checkout.js.
